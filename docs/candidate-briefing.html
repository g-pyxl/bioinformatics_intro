GitHub Basics — Lesson 1
Estimated time
45–60 minutes
Difficulty
Beginner
Welcome. This short exercise gives you a hands-on introduction to the GitHub workflow used in Clinical Bioinformatics teams every day. You will fix a bug in a Python script, commit your change, and open a Pull Request — all through the browser with no local software required. There are no trick questions and the task does not require previous knowledge or experience. We are looking for clear thinking, attention to detail, and the ability to follow technical instructions.

What you need

  • ✓
    A free GitHub account — create one at github.com/signup if you don't already have one. Use your real name so we can identify your submission.
  • ✓
    A modern web browser (Chrome, Firefox, Edge, or Safari). No local installation is required — the exercise runs entirely in the browser via GitHub Codespaces.

Getting started

Click the assignment link below to accept the exercise. GitHub Classroom will automatically create a private repository for you — only you and the instructor can see it.

Please create a new team (use your own name) and accept the assignment. Once you have accepted, click the repository link that appears. Your full step-by-step instructions are inside the repository at lessons/01_github_basics/README.md.

What you will do

1

Accept the assignment — GitHub Classroom creates your private repo.

2

Open a Codespace — a browser-based development environment loads automatically.

3

Create a branch — isolate your work from the main codebase.

4

Find and fix a bug — a Python script that calculates GC content of DNA sequences is producing wrong results. Read the output, identify the cause, and fix it.

5

Commit and push — save your fix with a clear, descriptive commit message.

6

Open a Pull Request — propose your fix for review. This is your submission.

7

Complete the quiz — a short multiple-choice quiz is linked at the end of the lesson.

What we are assessing

  • Ability to follow technical instructions
  • Logical debugging approach
  • Quality of commit message
  • Pull Request description
  • Attention to detail

Tips

  • Read the error output carefully before touching the code
  • The fix is a single line change
  • Write your commit message as if a colleague will read it
  • All instructions are inside the repo — follow them step by step
